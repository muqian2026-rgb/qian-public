# Issue Ready · Checklist

Copy into Issue comment or use as Agent audit table.

## Issue body

| Field | Ready? | Notes |
|-------|--------|-------|
| Context (why) | ☐ | |
| Goal (what) | ☐ | |
| Out of scope | ☐ | Explicit "not doing" |
| AC (testable) | ☐ | ≥3 bullets or GWT |
| Design link | ☐ | Frame ID or export path |
| Dependencies | ☐ | Linked issues or "none" |

## Architecture (TL)

| Field | Ready? | Notes |
|-------|--------|-------|
| Module boundary clear | ☐ | |
| API / data change noted | ☐ | |
| Known risks / spike done | ☐ | |
| **Blocker** open? | ☐ | Must be **no** for Ready |

## Product sign-off

| Field | Ready? | Notes |
|-------|--------|-------|
| PM confirms Ready for dev | ☐ | Comment or label |
| Master PRD § updated (if needed) | ☐ | Link to section |

## PR / DoD (for IC, post-Ready)

| Field | Done? | Notes |
|-------|-------|-------|
| PR links Issue | ☐ | |
| Review against AC | ☐ | |
| Tests per DoD | ☐ | |
| PRD updated if behavior changed | ☐ | |
