# 三层四面 · 商业分析

> **来源**：美团常用方法论（非面试流程）

## 是什么 / 不是什么

| 是 | 不是 |
|----|------|
| 判断市场机会、拆解收入增长的**分析表** | 面试答题套路 |
| 战略会 / 立项 / 月报 narrative 框架 | 供应链「三层」等同名概念 |

---

## 一、三层：值不值得做（外 → 内）

| 层 | 问什么 | 机会逻辑 |
|----|--------|----------|
| **1 行业大盘** | 整个蛋糕多大 | 大盘太小 → 谨慎 |
| **2 线上化率** | 互联网能切多少 | 线上化低 → 常有空间 |
| **3 公司市占率** | 我们能占多少 | 可抢份额 → 值得投入 |

```
好机会 ≈ 大盘大 + 线上化低 + 我们能抢份额
```

**输出**：三层表（指标 / 数值 / 来源 / 判断）。

---

## 二、四面：怎么增长

```
收入 ≈ 用户数 × 频次 × 客单价 × 货币化率
```

| 面 | 维度 | 常见指标 |
|----|------|----------|
| 人 | 用户数 | MAU、付费用户、新增用户 |
| 次 | 频次 | 人均年消费次、复购率 |
| 额 | 客单价 | GMV/订单量、均价 |
| 利 | 货币化率 | 抽佣率、毛利率、ARPU |

**GMV 涨跌必拆**：先问哪一「面」变了，再下结论。

---

## 三、何时用三层 vs 四面

| 场景 | 主用 |
|------|------|
| 新业务 / 新市场立项 | 三层 |
| 存量业务复盘、月报 | 四面 |
| 市场调研报告·市场容量章 | 三层 |
| MBR 解释收入变化 | 四面 |

---

## 四、执行步骤

1. **澄清场景**：机会评估（三层）还是增长拆解（四面）？
2. **取数**：有内部数据用内部；无则标注 `[估算]` + 来源。
3. **填表**：用 [references/industry-mapping.md](./references/industry-mapping.md) 问句骨架。
4. **结论**：三层给 go/谨慎/否；四面给「主驱动面」+ 1–3 条建议。
5. **命名区分**：若同时讨论供给链，标明「市场三层」≠ 其它「三层」术语。

---

## 五、负约束

- 不编造未给来源的市场数字
- 不替代决策者战略拍板
- 不混用同名「三层」术语而不加前缀
